Begining¶

At first you have only four commands available

wiki and help available for everyone

wiki¶

gives link to the current documentation

.wiki

help¶

shows available command for your current user permissions
it is changed also based on bot being connect ed to channel or not

possible usage:

list all root level commands:

.help

list commands from sub category beloning to base for example:

.help base

connect¶

initializes bot work for the current channel
attention!: this command and all further commands require having user permissions

  • being a server owner

  • or having permission to manage channels

  • or having role bot_controller

warning: default setting enables auto erasing of all messages older than 40 seconds in the connected channel!

.connect

disconnect¶

disconnects the bot from channel. It could be also used for mass setting nullification. Disconnect the bot and wait 10 seconds in order for all your current channel settings being gone.

.disconnect

Player Base Tracking¶

commands.base.attach_base(bot, storage) → discord.ext.commands.bot.Bot¶

base commands

Those are command to track status of player bases! Example of output below.

check the list of precised based names before using the commands

possible usages:

.base add MyStation

if your base have spaces in its name, better be using:

.base add "My Very Long Named Station"

you can add multiple bases in one command

. code-block:

.base add "My station #1", "My station #2", "My station #3"

also, you could add only sub string that the list of bases have

. code-block:

.base add Depot

it will render all bases having Depot in its name

additional commands

for deleting of one base name

. code-block:

.base remove Depot

to clear the whole list of bases

. code-block:

.base clear
